#### blogs目录

该目录为文档目录
所有文档都写在这个目录下。
文件夹不可为中文，否则路由返回会有问题

每篇文件可以通过 标签、分类等来便于分类，如下：
```md
 ---
title: 烤鸭的做法 // 文章的题目
date: '2019-08-08 08:00:00'//发布时间
sidebar: 'auto' // 是否开启侧边栏
categories: // 分类 （该分类不需要另外配置，直接写）
 - 烹饪
 - 爱好
tags: // 标签 标签不需要另外配置，直接写
 - 烤
 - 鸭子
keys: // 文章是否加密
 - '123456'
publish: false // 是否发布
---
---
```

### 发版步骤

由于github page的目录选择和vue-press之间有区别，无法选择打包后的public作为根目录文件。只能选择/root或者/docs目录
因此github page实际上选择的是根目录文件。
- 执行 npm run build
- 直接把打包后的public内容复制到上级(虽然目录会看起来乱，但是如果分其他分支处理，发版会很麻烦)。推送到github即可
